Introduction

Git est un outil de gestion de versions distribué. Il permet de suivre l’historique des fichiers, travailler en branches, collaborer à plusieurs et récupérer des versions précédentes d’un projet. GitHub est une plateforme en ligne qui héberge des dépôts Git, facilite la collaboration et permet le suivi des projets via des outils tels que les issues, les pull requests, ou les actions.

Cas d'utilisation simple

  • git init : Initialise un dépôt local
  • git remote add origin <lien du dépôt> : Ajoute le lien vers le dépôt distant
  • git add . : Ajoute tous les fichiers modifiés
  • git commit -m "message" : Crée un commit avec un message
  • git branch -M main : Renomme la branche principale en main
  • git push -u origin main : Pousse le projet sur GitHub

Glossaire des commandes utiles

  • git clone URL : Clone un projet distant
  • git status : Affiche l’état des fichiers
  • git checkout -b feature : Crée une branche
  • git merge feature : Fusionne une branche
  • git log : Affiche l’historique
  • git stash / git stash apply : Met de côté/restaure les modifications
  • git revert / reset : Annule ou modifie les commits
  • git remote -v : Liste les URL de dépôts distants
